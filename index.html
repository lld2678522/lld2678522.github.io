Selenium和Scrapy使用代理ip学习

Selenium 设置代理ip

google chrome

proxyStr = '--proxy-server=http://%s'% ip
options.add_argument(proxyStr)
driver = webdriver.Chrome(executable_path=driver_path, options=options)

PhantomJS

proxy = Proxy({
       'proxyType': ProxyType.MANUAL,
       'httpProxy': ip
 })
# 新建一个‘期望技能’？？？
desired_capabilities = DesiredCapabilities.PHANTOMJS.copy()
# 把代理ip加入到技能中
proxy.add_to_capabilities(desired_capabilities)
driver = webdriver.PhantomJS(executable_path=driver_path,                     desired_capabilities=desired_capabilities)

Scrapy设置代理ip

class DownloaderMiddleware(object):
    def process_request(self, request, spider):
        request.meta['proxy'] = ip

Scrapy解决代理ip失效问题

  • 创建Middleware类继承RetryMiddleware，重写process_exception方法

    class exMiddleware(RetryMiddleware):
    def process_exception(self, request, exception, spider):
        ## 针对超时和无响应的reponse,获取新的IP,设置到request中，然后重新发起请求
        print(str(exception))
        # ip失效时出现的异常
        if '10061' in str(exception) or '10060' in str(exception) 
            or 'Connection refused' in str(exception) or '113' in str(exception):
        # 重新获取你的ip
        ip = new_ip()

        if isinstance(exception, self.EXCEPTIONS_TO_RETRY) and not 
            request.meta.get('dont_retry', False):
        return self._retry(request, exception, spider)
  • 设置settings.py文件替换Middleware文件
DOWNLOADER_MIDDLEWARES = {
       'WeiBoTop500.middlewares.Weibotop500DownloaderMiddleware': 543,
       # 默认使用的异常Middleware类设置为不使用
       "scrapy.downloadermiddlewares.retry.RetryMiddleware": None,
       # 重写的异常Middleware类
       'WeiBoTop500.middlewares.exMiddleware': 544,
    }
  • 需要注意的 不同的python环境RetryMiddleware文件的位置有可能不同
    可以在scrapy启动日志中找到RetryMiddleware文件的位置
